    About Bog and Auk Languages

    Bog Language

    Purpose: Bog is designed for scripting game logic with a syntax inspired by natural elements.

    Keywords: bogfn, mire, sludge, peat, drain, suckin, fen, gurgle

    Data Types: num, dec, txt, bool

    Example:

    suckin "std.bog"

                bogfn main(): num {
                    fen count = 10
                    mire(count > 5) {
                        gurgle("Count is greater than 5")
                    } sludge {
                        gurgle("Count is 5 or less")
                    }
                    drain 0
                }

    Comments: Single-line comments use ~~, multi-line comments use ~~~ ... ~~~

    Auk Language

    Purpose: Auk is tailored for styling and designing game interfaces with a unique syntax differing from traditional CSS.

    Selectors:

    • .class - Select elements by class
    • #id - Select elements by ID
    • element - Select elements by tag name

    Properties: Auk introduces custom properties prefixed with fen- to define styles.

    Example:

    selector fen-color: #ff0000
    selector mire-background: #00ff00

    • Dynamic Styling Capabilities
